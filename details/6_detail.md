## 生活类比
图片就像照片墙上的照片：`alt` 文本是照片背面的标签（告诉盲人这张照片是什么），路径是照片的挂法（用图钉固定还是用胶带粘），图注是照片下方的说明文字。挂得好，照片墙赏心悦目；挂得乱，看着都累。

## 内容要点

### 1. 基础图片语法

- `![alt 文本](图片路径 "标题")`
- alt 文本的最佳实践（有意义但不冗余）
- title 属性（可选，鼠标悬停显示）

### 2. 图片链接

- `![图片](url)` → 只是图片
- `[![图片](url)](link)` → 点击图片跳转
- 嵌套语法解析

### 3. 路径策略

- 相对路径：`../assets/logo.png`（推荐，可移植）
- 绝对路径：`/assets/logo.png`（站根起点）
- 完整 URL：`https://example.com/logo.png`（CDN）
- 陷阱：Git 仓库中图片路径的正确设置

### 4. 图片尺寸控制（GFM 扩展）

- `![alt](url =100x200)`
- `![alt](url =100x)` 只限制宽度
- `![alt](url =x200)` 只限制高度

### 5. 图注

- GFM 原生不支持图注
- 替代方案：图片后面紧跟段落
- 通过 CSS 实现图注样式

## 设计取舍

- Q: 为什么 Markdown 没有原生图注？—— 保持简单，交给 CSS
- Q: 相对路径 vs CDN？—— 可移植 vs 性能
- Q: 为什么 alt 文本这么重要？—— 无障碍（WCAG）和 SEO

## 动手练习
1. 用三种不同的路径方式引用同一张图片，对比渲染结果
2. 尝试给 alt 文本留空 `![](url)`，观察无障碍渲染效果
3. 思考题：为什么 Markdown 不直接支持图片宽度属性？
