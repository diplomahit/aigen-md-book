# 第5章细纲：表格的艺术

## 标题
## 表格的艺术 —— 给数据建"网格"

难度: ★★★☆☆ (3/5)
预备知识: 第2章行内语法（表格中的行内格式化）

## 学习目标
1. 掌握 GFM 表格的基础语法和三种对齐方式
2. 理解表格中常见的陷阱和规避方法
3. 能使用 HTML 表格处理复杂场景

## 生活类比
Markdown 表格就像在草地上画网格——用 `-` 画横线、用 `|` 画竖线，网格画好后就可以在每个格子里放东西了。对齐方式就像给格子贴标签："这列是数字，左对齐还是右对齐？"

## 模块地图

```
GFM Table Spec
├── 基本语法：| 表头 | | |
├── 分隔线：|---|---|---|
├── 对齐方式：:--- | :---: | ---:
├── 单元格内的行内语法
└── 复杂表格的 HTML 兜底
```

## 内容要点

### 1. 基础表格

- 表头行 + 分隔线 + 数据行
- 分隔线中的 `-` 必须至少有一个
- 每行的列数必须一致

### 2. 三种对齐方式

- 左对齐：`:---`
- 居中对齐：`:---:`
- 右对齐：`---:`

### 3. 陷阱与注意事项

- 单元格中的 `|` 需要转义（`\|`）
- 表格中的粗体/斜体/链接等行内语法
- 空单元格的处理
- 表格前后必须有空白行（否则可能被识别为普通段落）

### 4. 复杂表格的 HTML 兜底

- Markdown 表格的局限性：合并单元格、colspan/rowspan
- 何时切换到 HTML `<table>`
- HTML 表格在 Markdown 中的兼容性

## 设计取舍

- Q: 为什么表格是 GFM 扩展而非 CommonMark 核心？—— 语法歧义太多，社区共识不足
- Q: 为什么不支持合并单元格？—— 纯文本中无法表达 colspan/rowspan
- Q: 为什么要用 `|` 分隔？—— 视觉清晰，源码可读性好

## 动手练习

2. 写一个三种对齐方式都有的表格
3. 思考题：为什么 Markdown 表格要求每行列数一致？
