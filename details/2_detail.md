# 第2章细纲：行内语法全解

## 标题
## 行内语法全解 —— 给文字戴上"首饰盒"

难度: ★★☆☆☆ (2/5)
预备知识: 无

## 学习目标
1. 掌握所有行内格式化语法（粗体、斜体、代码、链接等）
2. 理解转义字符的使用场景
3. 能识别并避免行内语法中的常见陷阱

## 生活类比
行内语法就像给文字戴首饰：粗体是钻戒（最醒目）、斜体是项链（低调修饰）、删除线是划痕（展示历史）、代码标记是标签（标识身份）、链接是名片（连接外部）。但戴太多首饰会显俗气——过度使用粗体和斜体会稀释强调效果。

## 模块地图

```
行内元素（CommonMark spec.md, 第4章）
├── 代码 span（反引号包裹）
├── 转义字符（\ 号）
├── 实体引用（&amp; &lt; &gt;）
├── 原始 HTML span
├── 粗体（**text** 或 __text__）
├── 斜体（*text* 或 _text_）
├── 链接 [text](url) 和 [text][ref]
├── 图片 ![alt](url)
└── 自动链接 <url>
```

## 内容要点

### 1. 代码标记

- 反引号语法：`code`
- 多行代码 vs 行内代码的区别
- 嵌套反引号的技巧（用两个反引号包一个反引号）

### 2. 粗体和斜体

- `**text**` 粗体、`*text*` 斜体
- `__text__` 粗体、`_text_` 斜体（下划线版本）
- `***text***` 粗斜体
- 嵌套规则：`**粗体里的*斜体*`
- 陷阱：下划线版本在变量名中的歧义（`_variable_name_`）

### 3. 链接

- 行内链接：`[文字](URL "标题")`
- 引用链接：`[文字][引用标识]` + `[引用标识]: URL "标题"`
- 自动链接：`<https://example.com>`
- 陷阱：URL 中的括号需要转义

### 4. 图片

- 语法：`![替代文字](URL "标题")`
- 图片链接：`[![图片](url)](链接)`
- alt 文本的重要性（无障碍 + SEO）

### 5. 删除线

- GFM 扩展：`~~删除的文字~~`
- 不在 CommonMark 核心规范中
- 适用场景：修订记录、价格变化

### 6. 转义和 HTML

- 转义字符 `\`：`\*` 输出文字星号
- HTML 标签混写：`<del>删除</del>`、`<sub>下标</sub>`
- 实体引用：`&lt;` `&gt;` `&amp;`

## 设计取舍

- Q: 为什么有 `**` 和 `__` 两套语法？—— 兼容性和灵活性
- Q: 为什么删除线不在核心规范？—— GFM 扩展，保持核心简洁
- Q: 为什么下划线语法有歧义？—— Markdown 无法完美兼容变量名

## 动手练习
1. 尝试在一个词上同时应用粗体和斜体，观察渲染效果
2. 写一个链接，URL 中包含括号 `#section(a)`，看看会发生什么
3. 思考题：Markdown 没有"上标"语法，如何用 HTML 实现？

## 本章小结
本章我们一起学习了行内语法——Markdown 的小巧思集合。从代码标记到粗斜体，从链接到图片，再到转义和 HTML 混写。
