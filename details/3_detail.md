# 第3章细纲：块级元素全解

## 标题
## 块级元素全解 —— 搭建文章的"骨架"

难度: ★★☆☆☆ (2/5)
预备知识: 无（或参考第2章行内语法）

## 学习目标
1. 掌握6级标题的语法和层级规则
2. 理解有序列表和无序列表的嵌套规则
3. 能正确使用引用块和水平分割线
4. 理解段落之间的空白行规则

## 生活类比
块级元素就像盖房子：标题是楼层标识（一楼、二楼），段落是房间（居住空间），列表是抽屉（分类收纳），引用是玻璃墙（透明的外部视野），分割线是承重墙（分隔不同功能区）。一个设计合理的布局，住起来才舒服。

## 模块地图

```
块级元素（CommonMark spec.md, 第2-3章）
├── 前置处理：换行、制表符规范化
├── 设置代码块（Indented code blocks）
├── 栅栏代码块（Fenced code blocks）
├── 标题（Headings）—— ATX 和 Setext
├── 段落（Paragraphs）—— 空白行分隔
├── 区块引用（Block quotes）—— > 符号
├── 列表（Lists）
│   ├── 无序列表（- * +）
│   └── 有序列表（1. 2. 3.）
├── 水平分割线（Thematic breaks）
└── 空行（Blank lines）
```

## 内容要点

### 1. 标题

- ATX 标题：`# H1` ~ `###### H6`
- Setext 标题：下划线式 `===` / `---`
- 陷阱：`### 标题 ###` 中的尾部 # 匹配规则
- 陷阱：`#` 后必须有空格，`#标题` 不是标题

### 2. 段落

- 段落之间需要一个空行分隔
- 段落内的换行：一个换行 vs 两个换行（空格+换行）
- 段落内的软换行（trailing spaces 技巧）

### 3. 列表

- 无序列表：`-` `*` `+` 三种标记符号等价
- 有序列表：`1.` `2.` `3.` 自动编号
- 列表嵌套：2个或4个空格缩进
- 陷阱：列表项之间的空行
- 陷阱：`1. ` 后面的空格是必须的

### 4. 引用块

- 单行引用：`> 文字`
- 嵌套引用：`>> 更深层`
- 引用中的列表、代码块

### 5. 水平分割线

- 语法：`---` `***` `___`（至少3个）
- 可以包含空格
- 陷阱：`--- `（后面没有内容）vs `---text`（不是分割线）

## 设计取舍

- Q: 为什么列表需要缩进？—— 区分列表项和段落
- Q: 为什么段落之间需要空行？—— 区分段落和代码块
- Q: 为什么三种无序列表符号等价？—— 灵活性

## 动手练习
1. 尝试在 `#` 后不加空格，观察渲染结果
2. 写一个嵌套了3层的有序列表，看看渲染效果
3. 思考题：为什么 Markdown 没有定义"居中对齐"？

本章我们学习了 Markdown 的块级元素——文章的结构骨架。从标题到段落，从列表到引用，再到水平分割线。
