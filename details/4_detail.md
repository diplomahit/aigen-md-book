# 第4章细纲：代码块与语法高亮

## 标题
## 代码块与语法高亮 —— 给代码穿上"防护服"

难度: ★★☆☆☆ (2/5)
预备知识: 第2章行内语法（行内代码 vs 代码块的区别）

## 学习目标
1. 掌握三种代码块语法（三反引号、三波浪线、缩进代码块）
2. 理解语言标识的作用和语法高亮的渲染流程
3. 能使用 GFM 的代码行号和高亮功能

## 生活类比
行内代码像"保鲜膜"——薄薄一层，保护短句；代码块像"保险箱"——厚实的容器，保护复杂的多行代码。语法高亮就像给保险箱贴上标签（"这是Python的保险箱"），让渲染器知道用什么颜色的钥匙去打开它。

## 模块地图

```
代码块（GFM spec）
├── 栅栏代码块（Fenced Code Blocks）
│   ├── 三反引号：```语言
│   ├── 三波浪线：~~~语言
│   └── 混合：混合使用反引号和波浪线
├── 缩进代码块（Indented Code Blocks）
│   └── 4空格或1Tab缩进
├── 语言标识（Info String）
│   └── ```javascript  → 语言识别
├── 代码行号（Folding / Line Numbers）
└── 行高亮（Line Highlighting）
```

## 内容要点

### 1. 三种代码块语法

- 三反引号（最常用）：```python
- 三波浪线（反引号嵌套时）：~~~python
- 缩进代码块（4空格/1Tab）

### 2. 语言标识

- ```javascript、```python、```go
- 语言别名：```js、```py、```ts
- 不带语言标识的代码块（纯文本）

### 3. 代码块中的转义

- 代码块开头和结尾的反引号数量必须大于内容中的最大连续反引号数

### 4. 语法高亮的实现

- Markdown 解析器只标注"语言"，不执行高亮
- 高亮由第三方库完成（Prism.js、highlight.js、Shiki）
- 解析流程：Markdown → AST（含语言标识）→ 高亮库 → HTML + CSS

### 5. GFM 扩展功能

- 代码行号：GitHub 自动显示
- 行高亮：```python {5} 高亮第5行

## 设计取舍

- Q: 为什么语法高亮不集成在 Markdown 规范中？—— 保持规范简洁，交给生态
- Q: 三反引号 vs 三波浪线？—— 反引号更常见，波浪线用于嵌套
- Q: 为什么缩进代码块不推荐？—— 难以阅读和维护

## 动手练习

2. 尝试用不存在的语言标识，看看渲染器如何处理
3. 思考题：为什么 Markdown 不直接支持行号？
